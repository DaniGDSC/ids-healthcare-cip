### Step 2: Create `phase5_explanation.py`
Single-file pipeline script (follows security_hardened pattern). Structure:

**Named constants:**
- `N_FEATURES = 29`, `TIMESTEPS = 20`, `STRIDE = 1`
- `BACKGROUND_SAMPLES = 100`, `TOP_K_FEATURES = 10`
- Explanation templates per risk level

**Functions (single-responsibility, in pipeline order):**

1. `_load_and_verify_artifacts()` — SHA-256 verify all Phase 2/3/4 artifacts via metadata JSONs
2. `_rebuild_model()` — reuse Phase 4 pattern: Phase 2 builders + Phase 3 head + load weights
3. `_load_risk_report()` — load risk_report.json, return sample_assessments list
4. `_filter_non_normal_samples()` — filter samples where risk_level != "NORMAL", log counts per level
5. `_prepare_background_data()` — 100 random Normal samples from train, reshape to (100, 20, 29)
6. `_prepare_explanation_data()` — windowed test data for non-NORMAL indices
7. `_compute_shap_values()` — `shap.GradientExplainer(model, background)` (TF2-compatible fallback from DeepExplainer), SHAP values shape (N, 20, 29)
8. `_aggregate_feature_importance()` — mean(|shap|) over timesteps → (N, 29), then global mean → (29,) ranked
9. `_enrich_sample_context()` — attach risk_level, timestamp, top 3 features + SHAP %, explanation string
10. `_generate_explanation_text()` — template-based per risk level
11. `_plot_waterfall()` — matplotlib waterfall chart for CRITICAL/HIGH samples
12. `_plot_feature_importance_bar()` — top 10 features, red=attack/blue=normal
13. `_plot_anomaly_timeline()` — anomaly score over timesteps with threshold crossing
14. `_export_artifacts()` — shap_values.parquet, explanation_report.json, explanation_metadata.json with SHA-256
15. `_generate_report()` — render §8.1 markdown with tables and chart references
16. `run_explanation_pipeline()` — main orchestrator
17. `main()` — entry point under `if __name__ == "__main__"`

**Key design decisions:**
- **SHAP method:** Try `shap.DeepExplainer` first; fall back to `shap.GradientExplainer` if custom layer causes issues
- **Feature aggregation:** SHAP values are (N, 20, 29); aggregate over timesteps via `mean(|shap|, axis=1)` → (N, 29) for per-feature importance
- **Waterfall charts:** Generate for first 5 CRITICAL + first 5 HIGH samples only (cap to avoid file explosion)
- **Timeline chart:** Use anomaly scores from risk_report.json across first 20 consecutive samples of an incident
- **Timestamps:** Simulated (Phase 4 doesn't store real timestamps) — use sample index as time proxy
- **HIPAA:** Never log individual patient feature values in explanations — only feature names and relative contribution percentages

## Risk Mitigation
- SHAP on 1616 samples with (20,29) input could be slow — cap at 200 samples if needed, sample stratified by risk level
- Custom BahdanauAttention layer may cause SHAP errors — GradientExplainer handles custom layers better than DeepExplainer
- Chart rendering on headless CI — use `matplotlib.use("Agg")` backend
